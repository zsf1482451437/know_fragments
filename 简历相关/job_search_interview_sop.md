# 前端求职投递与面试恢复 SOP

适用对象：半年以上未参加面试，准备重新投递前端 / React / React Native / 跨境商城相关岗位。

核心原则：

- 前 5 场面试优先恢复手感，不以拿 Offer 为唯一目标。
- 不要一开始投最想去的公司，先用中等匹配岗位校准表达和市场反馈。
- 投递分两条主线：`Web 前端` 和 `跨端 / React Native 前端`。
- 每次面试后必须复盘，否则面试经验不会沉淀。

固定使用以下文件：

- 主简历：`resume_zhaisifeng.md`

- 跨端版：`resume_zhaisifeng_cross_platform.md`

## 1. 岗位定位

### 优先投递

- React / Next.js / TypeScript 前端
- 跨境电商 / 独立站 / Shopify / SaaS
- React Native / 跨端 / App 内嵌 H5
- 前端性能优化 / SEO / 工程化
- 3-5 年前端开发

### 谨慎投递

- 纯 Android 原生
- 强 Java / Kotlin / Android Framework
- 强算法 / AI 模型训练
- 纯后端 Node.js
- 只要求 Vue 且不接受 React 背景

### 岗位匹配判断

看到 JD 后，按以下顺序判断：

1. 技术栈是否匹配：React / Next.js / TypeScript / React Native 是否出现。
2. 业务是否匹配：商城、电商、跨境、SaaS、App、内容运营是否出现。
3. 能力是否匹配：性能优化、组件化、SEO、多端适配、测试、工程化是否出现。
4. 风险是否过高：是否强要求 Java/Kotlin、Android Framework、原生架构经验。

匹配度低于 60% 的岗位不优先投。

## 7. 面试表达原则

### 先收束，再展开

推荐表达：

```text
我主要强项是 React/Next.js Web 前端和 React Native 跨端，Shopify 和 AI 工程化是业务场景里的实践补充。
```

避免表达：

```text
我 Web、移动端、后端、AI、架构都能做。
```

### 遇到不完全匹配岗位

推荐表达：

```text
Android 原生不是我的主方向，我更匹配的是 React Native、Web/H5、跨端融合、前端工程化和性能优化相关部分。
```

不要硬说自己精通 Java/Kotlin 或 Android Framework。

### 讲项目用 STAR

结构：

```text
背景：项目为什么做？
任务：我负责什么？
行动：我怎么做？
结果：结果如何，有什么数据？
```

示例：

```text
当时 GogoalShop 首页资源体积较大，LCP 在 4s 左右。我负责做首页性能优化，先用 Lighthouse 和 Performance 定位资源瓶颈，再通过代码分割、依赖治理、图片和字体优化处理首屏链路。最后关键 JS 从 3MB 降到 600KB，LCP 降到 2s 内。
```

## 12. 红线

不要做：

- 不看 JD 直接海投。
- 所有岗位用同一份简历。
- 面试前不复习对应项目。
- 回答时过度包装 Android 原生或 AI 能力。
- 把团队成果全部说成个人主导。
- 面试后不复盘。

必须做：

- 每天控制投递质量。
- 每场面试后记录问题。
- 每周根据反馈调整简历和话术。
- 对最想去的公司延后投递。
- 对不匹配岗位主动收束能力边界。

## 14. 一句话策略

先用 1 周恢复表达和市场反馈，再用 2-4 周集中投递目标岗位。  
Web 岗主打 `React/Next.js + 商城 + 性能 + SEO`。  
跨端岗主打 `React Native + 双端适配 + 测试 + 移动端性能`。  
遇到强 Android 原生岗位，主动强调跨端和 Web/H5 融合能力，不硬碰 Java/Kotlin 深水区。
