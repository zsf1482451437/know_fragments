## Why

该 JD 同时强调 React/TypeScript 基础、网络与工程化能力、AI Agent 开发经验，以及 Go、电商系统经验等加分项。围绕该 JD 建设一个 React + Go 的电商系统，可以把前端工程能力、全栈协作、业务建模和 AI 辅助研发流程沉淀成可展示、可面试复盘的项目资产。

当前简历与面试准备已有 React、RN、工程化和电商背景，但缺少一个能集中承载“React 生态 + Go 后端 + 电商业务 + Spec/TDD/Agent 工作流”的完整样板项目，因此需要以规格驱动方式先定义系统边界和验收标准。

## What Changes

- 新增一个面向 JD 展示的电商系统项目方案，前端使用 React 生态，后端使用 Go。
- 覆盖商品浏览、搜索筛选、购物车、下单、订单管理等核心电商链路。
- 体现前端最佳实践，包括 TypeScript 类型建模、组件拆分、状态管理、路由、接口层封装、错误与加载态处理。
- 体现工程化能力，包括 Vite 构建、代码规范、测试、Mock/联调策略和性能优化。
- 体现网络与浏览器基础，包括 HTTP API 设计、缓存策略、请求失败重试、首屏性能和资源加载优化。
- 体现 Go 后端能力，包括 REST API、领域模型、分层架构、数据持久化和基础并发/上下文控制。
- 体现 AI Agent 使用经验，将 OpenSpec/TDD/代码审查作为研发流程的一部分。

## Capabilities

### New Capabilities

- `ecommerce-product-discovery`: 商品列表、商品详情、搜索、筛选、排序与分页能力。
- `ecommerce-cart-checkout`: 购物车、库存校验、价格计算、提交订单和结算流程能力。
- `ecommerce-order-management`: 订单创建、订单列表、订单详情和订单状态流转能力。
- `react-frontend-engineering`: React + TypeScript 前端架构、组件体系、状态管理、路由和接口层工程化能力。
- `go-backend-service`: Go 后端服务、REST API、领域模型、持久化和错误处理能力。
- `agentic-development-workflow`: 基于 OpenSpec、TDD、AI Agent 和代码审查的研发流程能力。

### Modified Capabilities

- 无。

## Impact

- 新增电商系统的前后端项目结构与实现计划。
- 前端预计涉及 React、TypeScript、Vite、React Router、状态管理、测试框架和样式方案。
- 后端预计涉及 Go HTTP 服务、路由、中间件、领域服务、数据存储和 API 契约。
- 需要新增 OpenSpec 规格、设计文档和任务清单，作为后续实现与面试复盘依据。
- 不影响现有简历、题库和 job-tracker-app 数据；该变更先停留在规格与方案层，待确认后再进入实现。
