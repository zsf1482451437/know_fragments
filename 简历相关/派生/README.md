# 简历派生目录约定

后续基于 JD 派生出来的文件，统一放在当前目录：

- 定制化简历
- 匹配说明

默认约定如下：

1. 若未指定名称，简历文件名使用：

```text
resume_zhaisifeng_<company_or_role>_定制版.md
```

说明文件名使用：

```text
resume_zhaisifeng_<company_or_role>_匹配说明.md
```

2. 若明确指定“简历命名”，则直接使用该名称作为简历文件名主干，例如：

```text
平台工具前端-某公司.md
平台工具前端-某公司_匹配说明.md
```

3. 若用户直接给出完整输出路径，则以用户路径为准。
