# Boss 岗位归类 - 第一批

来源：`1.txt`，共 25 个岗位。

当前薪资基准：`18k`。

分档标准：

```text
适合练手：匹配度一般或薪资/岗位吸引力一般，适合恢复面试手感、练表达、验证简历。
适合去：技术栈和经历较匹配，薪资有提升空间，值得认真投递和准备。
想去：薪资、方向、成长性、简历匹配度都比较好，需要优先定制打招呼和面试准备。
```

## 总体结论

### 想去

优先级最高，建议先定制 Boss 招呼语和简历侧重点：

```text
8、21、3、2、12
```

### 适合去

匹配度较高，建议第二优先级投递：

```text
4、6、7、14、16、20、23
```

### 适合练手

用于恢复手感、测试市场反馈、练习高频问题：

```text
1、5、9、10、11、13、15、17、18、19、22、24、25
```

## 推荐投递顺序

### 第一优先级：重点定制

```text
21 -> 8 -> 3 -> 23 -> 20
```

原因：

```text
21：品牌和薪资最好，跨端/RN/Hybrid 方向匹配。
8：电商 + React + AI Agent + 数据可视化，和简历主线最贴。
3：AI 应用前端，适合把 AI skills 讲成真实工程流程。
23：RN 岗位，能消化你最近准备的实时预览、适配、长列表内容。
20：Web + RN + 性能优化，命中你的双主线。
```

### 第二优先级：正常投递

```text
2 -> 4 -> 14 -> 16 -> 7 -> 6 -> 12
```

原因：

```text
2：AI Coding 岗位薪资高，但需要冲刺准备，不能裸投。
4：AI Agent + 银行业务，方向好，但全栈追问重。
14：React/Next/Node 薪资空间大，可作为全栈潜力岗。
16：RN 岗位匹配，但原生调试要求较高。
7：AI + 工作流/低代码，方向可冲。
6：前端 + AI 工具 + Node，匹配但薪资一般。
12：你标记想去，但岗位偏初中级，作为保底想去。
```

### 第三优先级：练手投递

```text
9 -> 10 -> 22 -> 17 -> 18 -> 25 -> 11 -> 13 -> 1 -> 5 -> 24 -> 19 -> 15
```

原因：

```text
9/10/22：门槛适中，适合恢复手感。
17/18/25：可以练电商、组件化、性能、工程化表达。
11/13：泛前端岗位，可投但不必重准备。
1/5/24：要求偏高或技术栈偏离，谨慎投入。
19/15：核心方向偏 WebGL/Android，和当前主线不匹配。
```

## 每档投递策略

### 想去

策略：

```text
每个岗位单独定制 Boss 招呼语。
简历重点突出对应关键词：React、RN、Next.js、电商、AI Agent、工程化、性能优化。
面试前单独准备 3 个项目匹配点和 2 个风险解释。
```

优先准备内容：

```text
8：电商后台、数据可视化、埋点 SDK、AI 提效。
21：RN/Hybrid、小程序、运营后台、稳定性和性能。
3：AI Chat、Agent、知识库问答、流式输出、工作流。
2：AI skills、SubAgent、SDD/TDD、AI Code Review、全栈交付边界。
12：电商页面、UI 还原、性能优化、组件化。
```

### 适合去

策略：

```text
批量投递，但对 4、14、16、20、23 这些岗位做轻定制。
重点观察回复率和面试题方向，作为第二周主力投递池。
```

准备重点：

```text
4/14：补 Node.js、接口设计、数据库基础、全栈边界。
16/23：补 RN 原生桥接层、Native Module、适配、长列表、性能优化。
6/7/20：补工程化、AI 工具、跨端、低代码/工作流表达。
```

### 适合练手

策略：

```text
不用深度定制，主要用于拿面试、练节奏、验证简历问题。
遇到薪资明显低于 18k 或方向偏离太多的岗位，不投入过多精力。
```

练习目标：

```text
练自我介绍。
练 Reolink Web/RN 项目表达。
练 AI skills 真实性解释。
练 React/浏览器/工程化基础题。
记录被追问最多的问题，回头补进 P0/P1 文档。
```
