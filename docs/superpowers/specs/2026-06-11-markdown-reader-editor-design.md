# Markdown 阅读编辑器项目设计文档

## 1. 项目背景

当前 `know_fragments` 目录已经沉淀了大量 Markdown 知识文档，内容覆盖投资、前端、后端、工具、项目经验等多个主题。现阶段主要依赖 IDE 或文件管理器打开文档，适合编辑，但不适合以“知识库浏览器”的方式快速阅读、切换和轻量修改。

本项目目标是最小化实现一个本地 Markdown 阅读与编辑 Web App，让用户可以在浏览器中查看 `know_fragments` 文件树，打开 `.md` 文件，编辑内容并保存回原文件。首版不追求完整知识库平台能力，只解决“本地 Markdown 文件能被顺畅阅读和修改”的核心闭环。

## 2. 设计目标

### 2.1 核心目标

- 提供一个本地运行的 Web App，用浏览器作为 Markdown 文件阅读和编辑入口。
- 默认绑定 `/Users/bytedance/Desktop/项目/know_fragments` 作为工作区根目录。
- 支持浏览目录结构、打开 Markdown 文件、编辑文件内容、保存到本地磁盘。
- 保持实现简单、可维护、可快速迭代。

### 2.2 非目标

- 首版不实现全文搜索。
- 首版不实现新建、删除、重命名文件或目录。
- 首版不实现用户登录、权限系统、云同步、多端同步。
- 首版不实现富文本编辑器、双链、标签、图谱、发布站点等知识库高级能力。
- 首版不直接替代 Obsidian、Typora 或 IDE，只作为轻量本地阅读编辑入口。

## 3. 用户场景

### 3.1 日常阅读

用户启动本地服务后，在浏览器打开应用。左侧展示 `know_fragments` 的目录和 Markdown 文件，用户点击任意 `.md` 文件后，右侧加载文件内容，用于快速阅读。

### 3.2 轻量编辑

用户阅读文档时发现需要补充或修正内容，可以直接在编辑区修改 Markdown 原文。点击保存后，应用将内容写回本地对应文件。

### 3.3 知识库维护

用户可以在不频繁切换 IDE 文件树的情况下，快速在多个 Markdown 文档之间切换，完成轻量维护工作。

## 4. MVP 范围

### 4.1 必须实现

| 功能 | 描述 | 验收标准 |
| --- | --- | --- |
| 文件树浏览 | 展示 `know_fragments` 下的目录和 `.md` 文件 | 页面左侧能看到层级结构 |
| 打开文件 | 点击 `.md` 文件后读取内容 | 右侧编辑区显示文件原文 |
| 编辑内容 | 支持修改当前文件 Markdown 原文 | 输入内容能更新编辑区状态 |
| 保存文件 | 将当前编辑内容写回本地文件 | 刷新或重新打开后内容仍存在 |
| 保存状态 | 展示未保存、保存中、已保存、保存失败状态 | 用户能明确知道保存结果 |
| 路径安全 | 后端限制读写范围在工作区内 | 不能通过 `../` 读取根目录外文件 |

### 4.2 暂缓实现

| 功能 | 暂缓原因 |
| --- | --- |
| 全文搜索 | 会引入索引、性能和结果展示设计，非首版核心闭环 |
| 新建文件 | 需要补充命名校验和目录选择交互 |
| 删除文件 | 存在误删风险，需要回收站或确认机制 |
| 重命名文件 | 需要处理路径更新、冲突和打开状态同步 |
| Markdown 预览 | 有价值但不是保存闭环必需，可作为第二阶段增强 |
| 图片管理 | 涉及资源路径、上传、引用转换，首版不处理 |

## 5. 产品形态

### 5.1 页面布局

应用采用三段式布局：

| 区域 | 位置 | 内容 |
| --- | --- | --- |
| 文件树 | 左侧 | 目录、Markdown 文件、当前选中文件状态 |
| 顶部栏 | 右侧顶部 | 当前文件路径、保存按钮、保存状态 |
| 编辑区 | 右侧主体 | Markdown 原文编辑器 |

首版可以使用单页应用，不引入路由系统。文件选择状态保存在前端内存中，刷新页面后回到未选中文件状态即可。

### 5.2 交互流程

1. 用户启动本地服务。
2. 浏览器打开应用首页。
3. 前端请求文件树接口。
4. 后端扫描 `know_fragments` 工作区，返回目录和 `.md` 文件树。
5. 用户点击某个 `.md` 文件。
6. 前端请求文件读取接口。
7. 后端读取文件内容并返回。
8. 用户在编辑区修改内容。
9. 前端标记当前文件为未保存。
10. 用户点击保存。
11. 前端调用保存接口。
12. 后端校验路径并写入文件。
13. 前端展示已保存状态。

## 7. 后端设计

### 7.1 API 列表

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/files` | 获取 Markdown 文件树 |
| `GET` | `/api/file?path=relative/path.md` | 读取指定 Markdown 文件 |
| `POST` | `/api/file` | 保存指定 Markdown 文件 |
| `GET` | `/api/health` | 本地服务健康检查 |

### 7.2 文件树接口

请求：

```http
GET /api/files
```

响应：

```json
{
  "root": "know_fragments",
  "items": [
    {
      "type": "directory",
      "name": "AI重塑体系",
      "path": "AI重塑体系",
      "children": [
        {
          "type": "file",
          "name": "编码方案.md",
          "path": "AI重塑体系/ai/编码方案.md"
        }
      ]
    }
  ]
}
```

规则：

- 只返回目录和 `.md` 文件。
- 默认忽略 `.git`、`node_modules`、`.trae`、`.DS_Store`。
- 文件树按目录优先、名称升序排序。
- 路径统一使用相对工作区路径。

### 7.3 文件读取接口

请求：

```http
GET /api/file?path=AI重塑体系/投资/00_方法论/美股ETF/Day1_建立ETF地图_教学讲义.md
```

响应：

```json
{
  "path": "AI重塑体系/投资/00_方法论/美股ETF/Day1_建立ETF地图_教学讲义.md",
  "content": "# Day 1：建立 ETF 地图\n",
  "updatedAt": "2026-06-11T00:00:00.000Z"
}
```

错误：

- 文件不存在：返回 `404`。
- 路径越权：返回 `403`。
- 非 Markdown 文件：返回 `400`。

### 7.4 文件保存接口

请求：

```http
POST /api/file
Content-Type: application/json

{
  "path": "AI重塑体系/投资/00_方法论/美股ETF/Day1_建立ETF地图_教学讲义.md",
  "content": "# Day 1：建立 ETF 地图\n\n更新后的内容\n"
}
```

响应：

```json
{
  "path": "AI重塑体系/投资/00_方法论/美股ETF/Day1_建立ETF地图_教学讲义.md",
  "saved": true,
  "updatedAt": "2026-06-11T00:00:00.000Z"
}
```

保存规则：

- 保存前必须校验路径位于工作区内。
- 只允许保存 `.md` 文件。
- 首版直接覆盖原文件。
- 写入编码统一使用 `utf-8`。
- 保存失败时保留前端编辑区内容，不清空用户输入。

## 8. 路径安全设计

本项目虽然是本地工具，但后端具备文件读写能力，必须从首版就建立路径安全边界。

### 8.1 路径归一化

后端收到相对路径后执行以下步骤：

1. 拒绝空路径。
2. 拒绝绝对路径。
3. 使用 `path.resolve(workspaceRoot, relativePath)` 生成目标绝对路径。
4. 使用 `path.relative(workspaceRoot, resolvedPath)` 判断目标是否仍在工作区内。
5. 如果相对结果以 `..` 开头或为绝对路径，则拒绝访问。
6. 校验扩展名为 `.md`。

### 8.2 忽略目录

文件树扫描默认忽略：

```text
.git
node_modules
.trae
.DS_Store
dist
build
```

### 8.3 本地访问约束

首版服务默认只监听本机地址：

```text
127.0.0.1
```

不默认绑定 `0.0.0.0`，避免局域网设备访问本机文件读写服务。

## 9. 前端设计

### 9.1 状态模型

前端维护以下核心状态：

| 状态 | 类型 | 说明 |
| --- | --- | --- |
| `fileTree` | `FileNode[]` | 当前工作区文件树 |
| `selectedPath` | `string \| null` | 当前选中的 Markdown 文件路径 |
| `content` | `string` | 编辑区当前内容 |
| `originalContent` | `string` | 最近一次加载或保存成功后的内容 |
| `loadingTree` | `boolean` | 文件树加载状态 |
| `loadingFile` | `boolean` | 文件加载状态 |
| `saving` | `boolean` | 保存状态 |
| `error` | `string \| null` | 当前错误信息 |

是否未保存通过以下方式计算：

```ts
const dirty = content !== originalContent;
```

### 9.2 组件划分

| 组件 | 职责 |
| --- | --- |
| `App` | 管理页面整体状态和 API 调用 |
| `FileTree` | 渲染目录树，处理文件选择 |
| `EditorHeader` | 展示当前路径、保存按钮、保存状态 |
| `MarkdownEditor` | 编辑 Markdown 原文 |
| `EmptyState` | 未选择文件、加载失败等空状态 |

### 9.3 编辑器选择

首版推荐使用原生 `textarea`：

- 实现成本最低。
- 没有复杂依赖。
- 足够验证读写闭环。
- 后续可以平滑替换为 CodeMirror 或 Monaco。

首版 `textarea` 应具备：

- 等宽字体。
- 支持全高滚动。
- 保留换行和缩进。
- 输入时标记未保存状态。

## 10. 错误处理

### 10.1 后端错误

| 场景 | HTTP 状态码 | 前端提示 |
| --- | --- | --- |
| 路径为空 | `400` | 文件路径无效 |
| 非 Markdown 文件 | `400` | 只支持 Markdown 文件 |
| 路径越权 | `403` | 无权访问该路径 |
| 文件不存在 | `404` | 文件不存在或已被移动 |
| 读写失败 | `500` | 文件读写失败，请检查本地文件状态 |

### 10.2 前端错误

- 文件树加载失败时，左侧展示错误提示和重试按钮。
- 文件读取失败时，右侧展示错误提示，不覆盖当前已编辑内容。
- 保存失败时，保留编辑内容，并显示保存失败状态。
- 用户切换文件时，如果当前文件未保存，首版可以用浏览器 `confirm` 提醒是否放弃修改。

## 11. 测试策略

### 11.1 后端测试

重点测试：

- `pathGuard` 能阻止 `../` 越权路径。
- `pathGuard` 能阻止绝对路径。
- `pathGuard` 只允许 `.md` 文件。
- `fileTree` 只返回目录和 `.md` 文件。
- `fileStore` 能读取和保存临时 Markdown 文件。

### 11.2 前端测试

重点测试：

- 文件树节点点击后触发读取。
- 编辑内容后保存按钮可用。
- 保存成功后未保存状态消失。
- 保存失败时编辑内容不丢失。

### 11.3 手动验收

首版完成后执行以下验收：

1. 启动服务并打开浏览器页面。
2. 确认左侧能看到 `know_fragments` 下的 Markdown 文件。
3. 打开 `AI重塑体系/投资/00_方法论/美股ETF/Day1_建立ETF地图_教学讲义.md`。
4. 修改一行内容并保存。
5. 用 IDE 重新打开该文件，确认内容已写入。
6. 尝试访问 `../` 路径，确认被后端拒绝。

## 13. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 路径越权 | 可能读写工作区外文件 | 首版必须实现 `pathGuard` 并写测试 |
| 保存误覆盖 | 用户可能误改原文件 | 展示未保存状态，后续可增加自动备份 |
| 大目录扫描慢 | 首屏加载变慢 | 首版先全量扫描，后续改懒加载 |
| 大文件编辑卡顿 | `textarea` 体验下降 | 首版接受，后续升级 CodeMirror |
| 并发外部修改 | IDE 和应用同时编辑可能覆盖 | 首版不处理，后续可比较 `updatedAt` |

## 14. 成功标准

项目首版完成时，应满足：

- 用户可以通过一个本地命令启动服务。
- 浏览器页面能展示 `know_fragments` 的 Markdown 文件结构。
- 用户可以打开任意 `.md` 文件查看原文。
- 用户可以修改 Markdown 内容并保存回本地。
- 保存结果可以被 IDE 或文件系统验证。
- 后端无法读取或写入工作区外的文件。
