# Todoist 个人目标任务管理模板

## 目标

- 这份文档用于沉淀一套偏个人使用的 `Todoist` 目标任务管理模板。
- 核心不是把 Todoist 配得很复杂，而是建立一个轻量闭环：

```text
年度/季度目标 -> 本月重点 -> 本周推进 -> 今日执行
```

---

## 核心原则

- `Todoist` 只管行动项，不承载长文知识库。
- 项目用于分领域，标签用于横向过滤。
- 今日任务不超过 `3-5` 个。
- 每个目标都必须拆成下一步动作。
- 长文档、SOP、复盘沉淀放到 `Markdown / Notion / Obsidian`，Todoist 里只保留链接或下一步动作。

---

## 项目怎么分

建议使用“领域 + 执行层”的混合结构：

```text
1. 人生/年度目标
2. 本月重点
3. 工作推进
4. 学习成长
5. 生活事务
6. 等待/跟进
7. someday/想做
```

### 项目说明

| 项目 | 用途 |
|---|---|
| `人生/年度目标` | 放中长期目标，不放日常杂事 |
| `本月重点` | 承接本月最重要的 3-5 件事 |
| `工作推进` | 工作相关项目、交付物、会议后续 |
| `学习成长` | Web3D、AI、源码分析、SOP 沉淀 |
| `生活事务` | 缴费、购物、健康、出行、家事 |
| `等待/跟进` | 等别人回复、等结果、待确认事项 |
| `someday/想做` | 未来想做但近期不推进的事项 |

---

## 任务层级怎么用

建议用父任务承载“小目标”，子任务承载“具体动作”。

示例：

```text
项目：学习成长

父任务：完成 Web3D 第一阶段入门
- 子任务：搭建 web3d-harness-lab
- 子任务：完成 01-basic-scene
- 子任务：写实验结论
```

判断一个任务是否合格：

- 是否能在一个明确时间块内推进
- 是否有清晰完成标准
- 是否不是一句空泛目标

## 标签怎么打

标签只做横向过滤，不要和项目重复。

### 时间标签

| 标签 | 含义 |
|---|---|
| `@today` | 今天必须推进 |
| `@this_week` | 本周要推进 |
| `@quick` | 15 分钟内可完成 |
| `@deep` | 需要专注时间块 |

### 场景标签

| 标签 | 含义 |
|---|---|
| `@pc` | 需要电脑 |
| `@mobile` | 手机就能处理 |
| `@home` | 在家可做 |
| `@out` | 外出顺手可做 |

### 状态标签

| 标签 | 含义 |
|---|---|
| `@waiting` | 等待别人或等待结果 |
| `@next` | 当前项目的下一步动作 |
| `@important` | 高价值任务 |

### 最小标签集

建议一开始只保留这些：

```text
@today
@this_week
@deep
@quick
@next
@waiting
@important
@pc
```

不要一开始创建 20 个标签，否则维护成本会超过收益。

## 优先级怎么用

| 优先级 | 含义 |
|---|---|
| `P1` | 今天不做会明显影响目标推进 |
| `P2` | 本周重要 |
| `P3` | 普通待办 |
| `P4` | 备选或低优先级 |

建议规则：

- 每天 `P1` 不超过 `3` 个。
- 每周 `P1 + P2` 不超过真实产能。
- 不要把所有任务都打成 `P1`。

---

## 今日 / 本周 / 目标怎么串起来

### 1. 目标层

放在 `人生/年度目标`。

示例：

```text
2026 目标：系统掌握 Web3D 工程化能力
2026 目标：建立 AI 与工程方法论知识库
2026 目标：优化个人效率系统
```

### 2. 本月层

放在 `本月重点`，每月只保留 `3-5` 个重点。

示例：

```text
本月重点：完成 Web3D 学习路线前 2 个实验
本月重点：沉淀 3 篇 SOP 文档
本月重点：搭建稳定的个人任务管理系统
```

### 3. 本周层

给本周要推进的任务打 `@this_week`。

示例：

```text
[学习成长] 完成 01-basic-scene @this_week @deep @next P1
[学习成长] 整理 Web3D 材质实验提纲 @this_week @deep P2
[工作推进] 输出本周项目总结 @this_week @pc P2
```

### 4. 今日层

从 `@this_week` 里筛出当天真要做的，打上 `@today`。

示例：

```text
[学习成长] 搭建 Vite + Three.js 项目骨架 @today @deep @pc P1
[学习成长] 写 basic-scene 实验结论 @today @pc P1
[生活事务] 缴费 @today @quick P2
```

## 推荐筛选器

| 筛选器 | 规则 |
|---|---|
| `今天执行` | `@today` |
| `本周推进` | `@this_week & !@waiting` |
| `深度工作` | `@deep & !@waiting` |
| `等待跟进` | `@waiting` |
| `快速清空` | `@quick & !@waiting` |

## 日常使用 SOP

### 每天早上 5 分钟

1. 打开 `本周推进`。
2. 挑出今天最重要的 `3` 个任务。
3. 给它们打 `@today`。

### 白天执行时

1. 只盯 `今天执行`。
2. 新冒出来的想法先丢进对应项目。
3. 不要随意打乱今天清单。

### 每天晚上 5 分钟

1. 清掉完成项。
2. 没做完的任务决定继续 `@today` 还是退回 `@this_week`。
3. 给关键项目补一个新的 `@next`。

### 每周复盘 20 分钟

1. 回看 `人生/年度目标`。
2. 更新 `本月重点`。
3. 给下周任务打 `@this_week`。
4. 清理过期的 `@today` 和 `@waiting`。

## 示例任务库

```text
[人生/年度目标]
- 2026：系统掌握 Web3D 工程化能力

[本月重点]
- 完成 Web3D 前两个实验 @important P1

[学习成长]
- 完成 Web3D 第一阶段入门
  - 搭建 web3d-harness-lab @this_week @today @deep @pc @next P1
  - 完成 01-basic-scene @this_week @deep @pc P1
  - 写实验结论 @this_week @pc P2

[工作推进]
- 梳理当前项目 SOP @this_week @pc P2

[生活事务]
- 缴水电费 @today @quick P3

[等待/跟进]
- 等待合作方案回复 @waiting P2
```

---

## 不建议这样做

- 不要把标签当项目分类再用一遍。
- 不要把今天清单塞到 `10+` 个任务。
- 不要把目标写成无法执行的大词而不拆动作。
- 不要把所有任务都打成 `P1`。
- 不要把笔记全文塞进 Todoist。

---

## 最佳组合

如果你偏知识沉淀型个人管理，建议：

```text
Todoist：管执行动作
Markdown / Notion / Obsidian：管目标说明、SOP、知识库
```

一句话原则：

> Todoist 负责“下一步做什么”，知识库负责“为什么做、怎么沉淀”。
