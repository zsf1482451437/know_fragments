# 日历测试功能点清单

## 使用方式

- 状态标记：`[ ]` 待测，`[x]` 已测，`[~]` 需补测或存在疑问。
- 建议每次手测完成后，在对应条目后补充结论、截图或问题链接。
- 后续补测试时，优先从 `核心数据层`、`事件编辑器`、`事件详情`、`月视图/日视图导航` 开始。

## 核心数据层

- [ ] `CalendarStore.bootstrap` 初始化今天、聚焦月份、日历源和种子事件正确。
- [ ] `selectDate` 会归一化日期，并同步更新 `selectedDate` 与 `focusedMonth`。
- [ ] `eventsFor` 只返回当天事件，并受 `visibleCalendarIds` 过滤。
- [ ] `eventById` 能跨日期桶查找事件，找不到时返回 `null`。
- [ ] `createDraftEvent` 能创建事件、裁剪标题、归一化日期、同步选中日期和月份。
- [ ] `createDraftEvent` 能处理位置、备注、URL 为空字符串时归一化为 `null`。
- [ ] `createDraftEvent` 能保存全天、跨天、重复、提醒、邀请对象、日历来源等字段。
- [ ] `updateEvent` 能按 `id` 覆盖已有事件，而不是新增重复事件。
- [ ] `updateEvent` 修改日期后，事件会从旧日期桶移动到新日期桶。
- [ ] `updateEvent` 修改日历来源后，事件颜色跟随新日历来源更新。
- [ ] `updateEvent` 对可选字段支持清空和保留的边界行为符合预期。
- [ ] `deleteEvent` 能删除存在的事件，并在日期桶为空时清理桶。
- [ ] `deleteEvent` 删除不存在事件时返回 `false`，且不触发错误。
- [ ] `setCalendarVisibility` 能隐藏和恢复指定日历源。
- [ ] `hasEventsOn` 能正确反映当前可见日历下某天是否有事件。
- [ ] `setActiveView` 相同视图不重复通知，不同视图正确切换。
- [ ] `goToToday`、`goToPreviousMonth`、`goToNextMonth` 的日期和月份跳转正确。

## 模型与格式化

- [ ] `CalendarEvent.seed` 生成默认事件字段正确，包括默认时间、日历和颜色。
- [ ] `CalendarEvent.copyWith` 能修改普通字段，并能通过 sentinel 清空 `location`、`notes`、`url`。
- [ ] `EventAlertSetting` 相等性和 `hashCode` 支持提醒去重。
- [ ] `EventRepeatRule` 相等性和 `hashCode` 支持重复规则比较。
- [ ] `eventRepeatLabel` 覆盖从不、工作日、每天、每周、每月、每年、自定义间隔。
- [ ] `eventAlertLeadTimeLabel` 覆盖事件发生时、提前 5 分钟、提前 15 分钟、提前 1 小时、提前 1 天。
- [ ] `eventAlertSettingLabel` 覆盖出发时间、到达地点、离开地点等高级提醒。
- [ ] `eventAlertsSummary` 空提醒显示 `无`，多提醒按中文顿号连接。

## App 与导航

- [ ] App 使用 `CupertinoApp` 和 iOS 风格页面结构。
- [ ] `CalendarScope` 能向页面树提供同一个 `CalendarStore` 实例。
- [ ] 底部导航可在月视图、日视图、列表、收件箱之间切换。
- [ ] `CalendarAppShell` 根据 `activeView` 渲染正确页面。
- [ ] 页面切换时 `AnimatedSwitcher` 动画不导致状态错乱。
- [ ] 月视图顶部 `Calendars` 入口能进入日历管理页。
- [ ] 月视图顶部搜索按钮能进入搜索页。
- [ ] 日视图返回按钮能回到月视图。
- [ ] 事件卡片点击详情、详情页编辑、编辑后返回链路完整。

## 月视图

- [ ] 月视图显示当前聚焦月份和年份。
- [ ] 上一月、下一月按钮能正确切换月份，并选中目标月 1 日。
- [ ] 今天按钮能跳回固定今天日期。
- [ ] 月历网格按周一到周日显示。
- [ ] 月历网格能显示聚焦月份内所有日期，并保留空白占位。
- [ ] 当前选中日期有红色选中样式。
- [ ] 真实今天有红色今日样式。
- [ ] 有事件的日期显示事件点。
- [ ] 点击未选中日期只更新选中日期。
- [ ] 点击已选中日期会进入日视图。
- [ ] 顶部周条能显示选中日期所在周。
- [ ] 顶部周条点击日期后同步更新选中日期。

## 日程面板

- [ ] 日程面板显示选中日期的月份、日期和星期。
- [ ] 无日程时显示空状态 `没有日程`。
- [ ] 有日程时按事件列表展示标题、时间、位置和备注摘要。
- [ ] 全天事件在日程面板显示 `全天`。
- [ ] 点击日程面板日期区域进入日视图。
- [ ] 点击添加按钮打开新建事件页。
- [ ] 点击日程事件打开事件详情页。
- [ ] 日程列表切换日期时动画和内容同步正确。

## 日视图

- [ ] 日视图顶部显示年份、日期、星期。
- [ ] 滚动后顶部紧凑日期标签出现。
- [ ] 今天按钮跳回今天。
- [ ] 当前时间线只在今天显示。
- [ ] 全天区域无全天事件时显示 `没有全天事件`。
- [ ] 全天区域有事件时显示全天事件 chip。
- [ ] 时间轴展示 24 小时刻度。
- [ ] 普通事件按开始时间排序。
- [ ] 普通事件根据开始和结束时间计算垂直位置和高度。
- [ ] 短事件高度不低于最小可读高度。
- [ ] 点击普通事件卡片可展开和收起。
- [ ] 展开事件后显示位置、备注和 `查看详情`。
- [ ] 点击 `查看详情` 进入事件详情页。
- [ ] 切换选中日期时，展开状态清空。
- [ ] 切换到今天时，日视图自动滚动到当前时间附近。

## 事件详情

- [ ] 详情页能展示标题、颜色条、时间和星期。
- [ ] 全天事件详情显示 `全天`。
- [ ] 普通事件详情显示开始和结束时间。
- [ ] 有位置时展示位置区块，无位置时隐藏。
- [ ] 有重复规则时展示重复区块，无重复时隐藏。
- [ ] 有提醒时展示提醒区块，无提醒时隐藏。
- [ ] 有邀请对象时展示邀请对象区块，无邀请对象时隐藏。
- [ ] 有 URL 时展示 URL 区块，无 URL 时隐藏。
- [ ] 有备注时展示备注区块，无备注时隐藏。
- [ ] 日历区块显示事件所属日历名称。
- [ ] 点击编辑按钮打开带初始值的编辑页。
- [ ] 删除按钮弹出确认弹窗。
- [ ] 删除取消后事件仍保留在详情页。
- [ ] 删除确认后事件从 store 删除，并返回上一页。
- [ ] 如果事件已不存在，详情页能安全渲染空页面。

## 事件编辑器

- [ ] 新建事件页标题为 `新建事件`，编辑事件页标题为 `编辑事件`。
- [ ] 标题为空时保存按钮禁用。
- [ ] 输入标题后保存按钮可用。
- [ ] 新建事件保存后写入 store，并返回上一页。
- [ ] 编辑事件保存后按原 `id` 更新事件，并返回上一页。
- [ ] 标题、位置、URL、备注输入框能正确保存。
- [ ] 位置、URL、备注输入为空时能被归一化。
- [ ] 初次打开编辑页时能回填已有事件全部字段。
- [ ] 全天开关开启后隐藏开始时间和结束时间行。
- [ ] 全天事件保存为 `00:00 - 23:59`。
- [ ] 全天开关关闭后，结束时间不早于开始时间。
- [ ] 修改开始日期后，若结束早于开始，应自动修正结束时间。
- [ ] 修改开始时间后，若结束不晚于开始，应自动顺延 1 小时。
- [ ] 修改结束日期时，结束早于开始应被修正。
- [ ] 修改结束时间时，结束不晚于开始应自动顺延 1 小时。
- [ ] 日期选择器的取消按钮不会修改值。
- [ ] 日期选择器的完成按钮会提交选中值。
- [ ] 日历选择弹层能列出全部日历源。
- [ ] 切换日历源后，编辑器显示对应名称和颜色点。
- [ ] 保存后事件的 `calendarId` 和颜色正确。

- [ ] 选择 `自定义` 后显示间隔和单位设置。

- [ ] 自定义单位选择器可切换天、周、月、年。

- [ ] 点击完成后，编辑器重复行显示新规则。
- [ ] 保存事件后重复规则持久化到 store。

- [ ] 月份标签格式正确，例如 `5月`。

- [ ] `visibleWeekFor` 能返回选中日期所在完整一周。

- [ ] 跨月第一周和最后一周的空白占位正确。

## 视觉与交互回归

- [ ] 全局保持 iOS 原生风格，优先使用 Cupertino 组件。
- [ ] 表单采用 inset grouped 样式，日期和时间为独立行。
- [ ] 月视图、日视图、日程面板的间距、圆角、阴影保持一致。
- [ ] 底部导航安全区适配正确。
- [ ] 弹层、选择器、确认框在 iPhone 尺寸下无遮挡。
- [ ] 深色模式下主要文本、背景、分隔线和颜色点可读。
- [ ] 长标题、长备注、长 URL 不应导致布局溢出。
- [ ] 空状态页面视觉稳定，不应出现异常跳动。

## 建议补测试顺序

1. 核心数据层：先补 `CalendarStore` 和模型格式化的纯单元测试。
2. 事件编辑器：补新建、编辑、全天、时间修正、复杂字段保存的 Widget 测试。
3. 事件详情：补编辑入口、删除确认、条件区块显示隐藏。
4. 月视图和日程面板：补日期选择、进入日视图、添加事件、打开详情。
5. 日视图：补全天事件、普通事件排序、展开详情、切换日期清空状态。
6. 日历管理：补可见性过滤对各页面的联动。
7. 搜索、列表、收件箱：暂以占位页快照和导航测试为主。
