# 输入准备节点

## 范围说明

这里聚焦那些发生在主循环前、主要修改输入和上下文的节点。

---

## `generalizePrompt` 相关节点

### `add_tech_stack`

作用：

- 从 `question.techStack` 读取技术栈信息
- 生成技术栈说明
- 把说明追加到 `question.prompt`

它的核心价值是把“目标技术栈”显式写进需求。

### `detailed_prompt`

作用：

- 再次调用模型扩写需求
- 把原始题面改造成更具体的执行说明

它的核心价值是降低模型在主循环里的需求理解成本。

---

## `beforeEnterTrace` 相关节点

### `add_thinking_prompt`

作用：往 `messages` 中插入 thinking 规则，稳定模型在 Agent 模式下的输出格式。

### `use_question_messages`

作用：在承接历史消息或上游产线结果时，把已有 messages 注入当前上下文。

### `replace_work_dir`

作用：把 prompt 中的抽象工作目录替换成真实沙箱 workDir。

### `download_zip_repo`

作用：在某些非 Git 场景下，直接把 zip 形式的代码快照拉到本地。

### `inject_run_playwright_test_tool`

作用：在真正执行前向模型暴露额外工具，让主循环中的工具集合更完整。

---

## 小结

这一类节点的共同特征是：

- 不直接交付结果
- 但会显著影响后续主循环的稳定性和正确性

可以把它们看作整个执行链的“起跑线整理器”。
