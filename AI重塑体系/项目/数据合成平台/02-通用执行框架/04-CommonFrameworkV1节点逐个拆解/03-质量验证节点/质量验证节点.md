# 质量验证节点

## 范围说明

这里聚焦主循环结束后的启动、校验与测试相关节点。

---

## `startService` 相关节点

### `flutter`

- Flutter Web：构建并启动 `build/web`
- Flutter Android：执行 APK 构建

### `vite` / `nextjs` / `webpack` / `html` / `auto` / `expo` / `uniapp-h5`

作用：按目标框架决定启动方式。

---

## `commonChecker` 相关节点

### `dart_analyze`

- 执行静态分析
- 统计错误与警告
- 折算 signal 分数

### `flutter_build`

- 再做一次真实 build
- 验证最终可构建性

### `style_check` / `click_check`

- 前者偏样式质量
- 后者偏交互检查

### `echarts_render_check` / `chartjs_render_check` / `d3js_render_check` / `animation_render_check`

- 面向可视化和动画场景做专项渲染验证

---

## `genUnitTest` 相关节点

### `flutter_widget_test_gen`

作用：探索仓库、生成测试、真实执行、根据错误迭代修复，直到得到可运行脚本。

### `gen-playwright-by-conversation` / `gen-playwright-by-explore`

作用：基于对话或代码探索结果生成 Playwright 测试。

### `use-question-unit-checkers` / `gen-visualization-test`

作用：复用已有测试描述或为特定场景生成专项测试。

---

## `runUnitTest` 相关节点

### `run_flutter_test`

作用：正式写入 Flutter 测试文件并执行 `flutter test`，计算通过率与 signal。

### `run_playwright_test` / `run_rubrics` / `agent_verify` / `trajectory_judge` / `check_uniapp_mp_build`

作用：分别从自动化测试、规则评估、验证模型、轨迹判断和专项构建检查等角度做后置验证。

---

## 小结

这类节点共同回答一个问题：

- 模型说“做完了”，平台如何用真实运行与质量校验证明它真的做完了。
