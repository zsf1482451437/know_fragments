# 沙箱初始化流程

## 主题定位

这一节聚焦 `createMarsSandboxSession()`，解释平台如何申请远端执行环境。

---

## 输入来源

平台会读取：

- `sandbox.sourceToken`
- `sandbox.source`
- `sandbox.image`
- `sandbox.spec`
- `sandbox.ttl`
- `pipelineType`
- `nfcVersion`

这些字段共同决定沙箱的来源、规格、镜像和有效时长。

---

## Mars 与 VKE 的差异

### Mars

- 通过 `SANDBOX_DOMAIN` 和 `SANDBOX_TRAE_DOMAIN` 接入
- 使用 `mark = pipelineType` 申请会话

### VKE

- 通过 token 向 `aidp-lite` 侧申请
- 按 `spec/image/duration/version` 创建实例

因此 `sandbox.source` 本质上是运行环境来源的总开关。

---

## 会话初始化后的处理

申请成功后，平台会：

- 把 session 写入 `state.marsSandboxSession`
- 记录到 `state.extra.session`
- 额外等待一小段时间，给环境稳定窗口

---

## 小结

没有沙箱初始化，就没有真实运行现场；后面的 Git、模型、工具、构建和测试都无法发生。
