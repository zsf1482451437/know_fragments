# 初始化命令与时序依赖

## 主题定位

这一节聚焦 `initGitEnv()` 后半段以及它与 `initConversation()` 的时序关系。

---

## 平台默认初始化命令

clone 完仓库后，平台会先执行一批默认命令，例如：

- 配置 npm registry
- 安装 `create-vite`
- 安装 `http-server`
- 在需要时安装并初始化 pnpm

这些命令属于平台级环境准备。

---

## 模板自定义初始化命令

在默认命令之后，会执行模板里的 `initCommandList`。

这让每个模板可以根据场景叠加自己的初始化逻辑，例如：

- 清空工作区但保留 `.git`
- 预装依赖
- 做环境变量预处理

---

## 为什么 prompt 构造必须放在 Git 初始化之后

`initConversation()` 需要读取当前目录结构并放进 prompt。

因此顺序必须是：

```text
申请沙箱 -> 初始化仓库 -> 读取目录 -> 构造对话
```

如果在 Git 初始化前构造 prompt，模型看到的目录就不是真实现场。

---

## 小结

这部分体现了平台一个非常工程化的原则：

- 先把运行现场准备好，再让模型开始工作。

这让模型的规划与真实代码仓上下文保持一致。
