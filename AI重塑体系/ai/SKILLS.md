# Skills

这个文件用于持续收录值得复用的 skill，方便后续查阅、比较和扩展。

## 收录规则

- 每个 skill 记录名称、分类、核心能力、适用场景和推荐理由。
- 描述尽量简洁，优先写清楚“为什么值得保留”。
- 新增 skill 时，按相同格式继续追加即可。

## Skill List

### 1. superpower

- 分类：工程型。
- 核心能力：TDD（测试驱动开发）能力强。
- 适用场景：适合需要先补测试、边写边验证、降低回归风险的开发任务。
- 推荐理由：在需求不够稳定或改动影响面较大时，先通过测试定义行为，再推进实现，会更稳。
- 备注：适合作为优先考虑的工程型 skill 之一。
