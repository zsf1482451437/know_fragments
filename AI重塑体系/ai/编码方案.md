# OpenAI / Google / Anthropic 模型对比

更新时间：`2026-05-12`

## 结论

- 如果只用一句话概括：`OpenAI` 更偏最强 agent / 自动化执行，`Google` 更偏价格与能力比，`Anthropic` 更偏代码与生产稳定性。
- 如果只看旗舰综合能力，当前最值得横向比较的是：`GPT-5.5`、`Gemini 3.1 Pro`、`Claude Opus 4.6`。
- 如果更看重性价比而不是天花板，优先关注：`Gemini 3.1 Pro`、`Claude Sonnet 4.6`、`GPT-5.4`。
- 下文价格统一按 `API 标准价 / 1M tokens / USD` 统计，不含 `Batch`、缓存、企业折扣和特殊区域价格。
- “准确度”这里拆成两层：一层是各模型的代表 benchmark，一层是旗舰统一横评。原因是第二、第三档模型不一定有完全一致的公开统一榜单。

## 旗舰准确度横评

| 旗舰模型 | 综合能力 | 代码 / Agent | 价格感受 | 结论 |
|---|---|---|---|---|
| `GPT-5.5` | 很强，OpenAI 公开 benchmark 基本都在前排 | `Terminal-Bench 82.7%` 明显强，适合代理执行、工具链、长流程任务 | 贵，尤其输出贵 | 要最强执行能力，它属于第一梯队 |
| `Gemini 3.1 Pro` | `AA 指数 57`，和 GPT-5.4 同档；`GPQA 94.3%` 很亮眼 | 代码也强，但更突出的是推理、多模态和长上下文 | 同级旗舰里很划算 | 三家里价格 / 能力比最漂亮 |
| `Claude Opus 4.6` | 综合很顶，但更偏稳和深 | 长代码库、复杂知识工作、深搜索任务非常强 | 不便宜，但比历史上的 Opus 便宜很多 | 高价值任务值得上，默认不一定最省钱 |

## 选型建议

### 1. 想要最强 agent / 自动化执行

- 首选：`GPT-5.5`
- 原因：在 `Terminal-Bench`、`GDPval`、工具使用和长流程执行上表现非常突出。

### 2. 想要旗舰里最强性价比

- 首选：`Gemini 3.1 Pro`
- 原因：价格低于同级很多旗舰，推理、多模态、长上下文能力很强。

### 3. 想要默认生产模型

- 首选：`Claude Sonnet 4.6`
- 备选：`GPT-5.4`
- 原因：这两档都属于“够强、够稳、价格还能接受”的默认生产位。

### 4. 想要高并发、低成本

- 首选：`Gemini 3.1 Flash-Lite`
- 备选：`Claude Haiku 4.5`
- 原因：两者都适合分类、抽取、轻量问答、批处理和高 QPS 服务。

### 5. 想要超长文档、多模态、超大上下文

- 首选：`Gemini 3.1 Pro`
- 原因：Google 在这类任务上的上下文和多模态路线更有优势。

### 6. 想要大代码库任务、代码审查、复杂多步知识工作

- 首选：`Claude Opus 4.6`
- 备选：`Claude Sonnet 4.6`
- 原因：Anthropic 在大代码库理解、长任务稳定性和生产可用性上依然非常强。

## 三家公司对应的编辑器 / 编码工具

这里的“编辑器”不只指传统 GUI IDE，也包含三家公司官方主推的编码入口，例如 `CLI`、`IDE 插件`、`桌面工作台` 和 `Web 编码界面`。

| 公司 | 官方主力产品 | 形态 | 说明 |
|---|---|---|---|
| OpenAI | `Codex CLI` | 终端 / CLI | OpenAI 当前最明确的官方编码入口，主打在终端里读仓库、改文件、跑命令、做 agentic coding |
| OpenAI | `Codex App` | 桌面工作台 / App | 更像带可视化工作流的编码工作台，适合不想全程待在纯终端里的用户 |
| OpenAI | `Codex IDE 集成` | IDE 插件 / 编辑器内集成 | 官方文档与仓库已明确支持把 Codex 放进 `VS Code`、`Cursor`、`Windsurf` 一类编辑器中使用 |
| Google | `Gemini Code Assist` | IDE 插件 / 编码助手 | Google 官方编码助手产品，主打在 IDE 中补全、问答、重构与代码理解 |
| Google | `Gemini CLI` | 终端 / CLI | Google 官方开源终端代理，适合在命令行里做代码理解、文件操作、命令执行和排障 |
| Google | `Gemini in Android Studio` | 专用 IDE 集成 | 面向 Android 场景的官方深度集成，适合 Compose、Gradle、Crash 分析等 Android 开发任务 |
| Anthropic | `Claude Code` | 终端 / CLI | Anthropic 最核心的官方编码产品，强调直接在代码库里工作、改文件、跑测试、做多步任务 |
| Anthropic | `Claude Code for VS Code / JetBrains` | IDE 插件 | Claude Code 已提供 `VS Code` 和 `JetBrains` 原生扩展，适合喜欢在 IDE 内看 diff 和审查修改的开发者 |
| Anthropic | `Claude Code Desktop / Web` | 桌面 / Web 编码工作台 | Claude Code 也可在桌面应用和网页中发起编码任务，更适合并行任务和远程委派 |

### 简单理解

- `OpenAI`：更像把 `Codex` 做成一个跨终端、桌面和编辑器的统一 coding workspace。
- `Google`：更像把 `Gemini` 深度嵌进现有 IDE 体系，尤其强调 `Gemini Code Assist` 和 `Android Studio`。
- `Anthropic`：更像以 `Claude Code` 为核心，先占据终端，再延伸到 `VS Code`、`JetBrains`、桌面和 Web。

### 如果按开发习惯来选

- 习惯终端流：优先看 `Claude Code` 或 `Codex CLI`
- 习惯 IDE 内完成开发：优先看 `Gemini Code Assist` 或 `Claude Code` 的 IDE 扩展
- 做 Android 开发：优先看 `Gemini in Android Studio`
- 想把编码、代理执行、桌面工作流放在一套系统里：优先看 `Codex`

## 备注

- OpenAI 这里将 `GPT-5.5 / 5.4 / 5` 视作“三个主力前排模型”。如果后续要补 `o3 / o4-mini` 这类偏推理系模型，可以单独再开一节。
- Anthropic 的官方定价文档在部分区域会直接跳转到不可访问页面，因此本文对 `Opus 4.6 / Sonnet 4.6 / Haiku 4.5` 的价格优先采用官方发布页与公开说明交叉确认。
- 不同 benchmark 的测试方法差异很大，不能把单一分数直接当成真实业务效果。真正落地时，仍建议基于自身任务做小规模评测。
